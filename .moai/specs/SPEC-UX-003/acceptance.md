# SPEC-UX-003: 인수 조건

## 시나리오

### AC-001: Pivot 모드 기본 진입

**Given** 신규 사용자가 첫 로그인할 때
**When** 메인 화면이 표시되면
**Then** Pivot 레이아웃이 표시되어야 한다 (사이드바 + 메인뷰)
**And** 사이드바에 카테고리 5개, 태그 상위 15개가 보여야 한다

### AC-002: 사이드바 카테고리 선택

**Given** Pivot 모드에서 사이드바에 "AI 도구" 카테고리가 보일 때
**When** 사용자가 클릭
**Then** 메인 뷰에 해당 카테고리의 북마크만 표시되어야 한다
**And** ContextHeader에 "📂 AI 도구"가 표시되어야 한다

### AC-003: 태그 단일 선택

**Given** 사이드바 태그 목록에 "ai"가 있을 때
**When** 사용자가 "ai" 클릭
**Then** 메인 뷰에 ai 태그를 가진 모든 북마크가 표시되어야 한다

### AC-004: 카테고리 + 태그 AND 필터

**Given** "AI 도구" 카테고리 + "video" 태그가 동시 선택된 상태
**When** 메인 뷰가 갱신되면
**Then** 두 조건을 모두 만족하는 북마크만 표시되어야 한다

### AC-005: 검색 입력

**Given** 메인 뷰에 100개 북마크가 표시된 상태에서
**When** 검색 입력에 "chat" 입력
**Then** name 또는 url에 "chat"이 포함된 북마크만 남아야 한다 (debounce 100ms 후)

### AC-006: 즐겨찾기 토글

**Given** 북마크 행에 ⭐ 아이콘이 표시된 상태에서
**When** 사용자가 클릭
**Then** 해당 북마크의 favorite 플래그가 토글되고 아이콘이 채워짐/비워짐
**And** "즐겨찾기" 컨텍스트에서 해당 북마크가 즉시 표시/숨김 처리

### AC-007: 격자/리스트 뷰 전환

**Given** 메인 뷰가 list 모드일 때
**When** 사용자가 ⊞ 격자 토글 클릭
**Then** 동일 데이터가 격자 카드 형태로 재렌더링되어야 한다

### AC-008: 밀도 변경

**Given** 메인 뷰가 list 모드 + comfortable 밀도일 때
**When** 사용자가 Compact 선택
**Then** 행 높이가 40px로 즉시 변경되어야 한다 (가상화 리스트 itemSize 갱신)

### AC-009: 가상화 리스트 성능

**Given** 1000개 북마크 시드 데이터
**When** 사용자가 스크롤
**Then** 60fps 유지되어야 한다 (DevTools profiler 검증)
**And** DOM에 최대 ~30개 행만 mount 되어야 한다

### AC-010: Favicon 표시 + fallback

**Given** 북마크 URL이 유효한 도메인일 때
**When** 행이 렌더링되면
**Then** Google s2 favicon이 16-32px로 표시되어야 한다
**And** 이미지 로드 실패 시 도메인 첫 글자 원형이 표시되어야 한다

### AC-011: 자주 쓰는 것 표시 (전체 컨텍스트)

**Given** 사용자의 usageStore에 5개 이상 사용 기록이 있을 때
**When** 사이드바 "전체" 클릭
**Then** 메인 뷰 상단에 "자주 쓰는 것" 8개가 격자로 표시되어야 한다
**And** 그 아래 전체 북마크 리스트가 이어져야 한다

### AC-012: 빈 상태 안내

**Given** 카테고리 선택했는데 매칭 북마크 0건
**When** 메인 뷰가 표시되면
**Then** "이 카테고리에 북마크가 없습니다" 메시지 + "+ 북마크 추가" 버튼 표시

### AC-013: 사이드바 접기

**Given** 사이드바가 펼쳐진 상태
**When** 사용자가 토글 버튼 클릭
**Then** 사이드바가 60px로 축소되고 아이콘만 표시되어야 한다
**And** 메인 뷰가 가용 너비를 채워야 한다

### AC-014: 키보드 j/k 네비게이션

**Given** 메인 리스트에 항목이 있고 첫 항목이 선택된 상태
**When** 사용자가 `j` 누름
**Then** 두 번째 항목이 강조 선택되어야 한다
**And** Enter로 해당 URL 새 창 열기

### AC-015: 키보드 1-9 카테고리 이동

**Given** 사이드바에 카테고리 5개가 있을 때
**When** 사용자가 `3` 누름
**Then** 3번째 카테고리가 컨텍스트로 선택되어야 한다

### AC-016: 우클릭 컨텍스트 메뉴 (선택 사항 -- 1차에서는 dropdown 버튼)

**Given** 북마크 행에서
**When** 사용자가 우클릭 (또는 ⋯ 버튼 클릭)
**Then** 편집/삭제/태그 추가/카테고리 이동 메뉴가 표시되어야 한다

### AC-017: Breadcrumb 클릭 해제

**Given** ContextHeader가 "📂 AI 도구 > 🏷 #ai" 일 때
**When** 사용자가 "🏷 #ai" 클릭
**Then** 태그 필터가 해제되고 카테고리만 남아야 한다

### AC-018: ARIA 접근성

**Given** 사이드바 트리와 메인 리스트
**When** axe-core 자동 검증 실행
**Then** ARIA tree / listbox 패턴 violation 0 이어야 한다

## 엣지 케이스

### EDGE-001: 카테고리 0개 사용자

**Given** 신규 사용자로 카테고리/북마크 없음
**When** Pivot 진입
**Then** 빈 상태 안내 + "+ 북마크 추가" 가이드 표시

### EDGE-002: 태그 1개도 없을 때

**Given** 모든 북마크가 tags 빈 배열
**When** 사이드바 렌더링
**Then** 태그 섹션은 "태그가 없습니다" 또는 hidden 처리

### EDGE-003: 매우 긴 카테고리/태그 이름

**Given** 100자 카테고리명
**When** 사이드바 렌더링
**Then** 말줄임으로 잘려 표시 + tooltip으로 전체 표시

### EDGE-004: 좁은 화면 (768px 미만)

**Given** 브라우저 너비 600px
**When** Pivot 진입
**Then** 사이드바 자동 collapsed 모드
