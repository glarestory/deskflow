# SPEC-IMPORT-001: 인수 테스트

## 시나리오 1: 정상 북마크 파일 가져오기 (병합 모드)

**Given** 사용자가 로그인되어 있고 기존 북마크 카테고리가 존재하는 상태에서
**When** TopBar의 "+ 가져오기" 버튼을 클릭하고
**And** Chrome에서 내보낸 유효한 HTML 파일을 선택하고
**And** 미리보기에서 파싱된 카테고리와 링크 수를 확인하고
**And** 병합(merge) 모드를 선택하여 확인하면
**Then** 파싱된 카테고리가 기존 북마크에 추가되고
**And** 각 카테고리에 이모지 아이콘이 자동 할당되고
**And** bookmarkStore를 통해 저장이 완료된다

## 시나리오 2: 정상 북마크 파일 가져오기 (전체 교체 모드)

**Given** 사용자가 로그인되어 있고 기존 북마크 카테고리가 존재하는 상태에서
**When** 유효한 Chrome 북마크 HTML 파일을 가져오기하고
**And** 전체 교체(replace) 모드를 선택하여 확인하면
**Then** 기존 북마크가 모두 삭제되고
**And** 가져온 북마크로 전체 교체된다

## 시나리오 3: 중복 카테고리 병합

**Given** 기존 북마크에 "Work" 카테고리(링크 3개)가 있는 상태에서
**When** "Work" 폴더(링크 2개)가 포함된 HTML 파일을 병합 모드로 가져오면
**Then** "Work" 카테고리가 중복 생성되지 않고
**And** 기존 "Work" 카테고리에 새로운 링크 2개가 추가되어 총 5개 링크가 된다

## 시나리오 4: 잘못된 파일 선택

**Given** 가져오기 모달이 열린 상태에서
**When** 북마크 형식이 아닌 일반 HTML 파일을 선택하면
**Then** 오류 메시지("올바른 Chrome 북마크 파일이 아닙니다" 등)가 표시되고
**And** 가져오기가 진행되지 않는다

## 시나리오 5: 빈 카테고리 필터링

**Given** Chrome 북마크 HTML에 링크가 없는 빈 폴더가 포함되어 있을 때
**When** 파일을 파싱하면
**Then** 빈 폴더는 미리보기에 표시되지 않고
**And** 가져오기에서도 제외된다

## 시나리오 6: 이모지 자동 할당

**Given** Chrome 북마크에 "개발 도구", "News Feed", "쇼핑몰" 폴더가 있을 때
**When** 파일을 파싱하면
**Then** "개발 도구"에 💻, "News Feed"에 📰, "쇼핑몰"에 🛍️ 이모지가 할당되고
**And** 매핑되지 않는 폴더에는 기본 이모지 🔖가 할당된다

## 엣지 케이스

### EC-001: 대용량 파일

**Given** 수백 개의 북마크가 포함된 대용량 HTML 파일을 선택했을 때
**When** 파싱이 진행되면
**Then** 로딩 인디케이터가 표시되고
**And** UI가 멈추지 않으며 파싱이 완료된다

### EC-002: 비정상 HTML 구조

**Given** 일부 태그가 누락되거나 비표준인 HTML 파일을 선택했을 때
**When** 파싱을 시도하면
**Then** 크래시 없이 파싱 가능한 부분만 결과에 포함된다

### EC-003: 파일 선택 취소

**Given** 가져오기 모달에서 파일 선택 대화상자가 열린 상태에서
**When** 사용자가 취소를 누르면
**Then** 모달이 파일 선택 단계에 그대로 유지된다

### EC-004: 중첩 폴더 구조

**Given** Chrome 북마크에 다단계 중첩 폴더(폴더 > 하위폴더 > 링크)가 있을 때
**When** 파싱하면
**Then** 하위폴더도 독립적인 카테고리로 변환된다
